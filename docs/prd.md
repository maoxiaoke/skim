# Skim (`skim`) — Skills 管理器 · PRD

> 阶段：Scrum Phase 3（产品经理产出）｜ 日期：2026-06-12 ｜ 状态：待评审

## 1. 产品概述

**一句话**：Skim 是一个跨 agent 的本地 AI 技能（Skills）管理桌面应用，让 Claude Code / Codex 用户安全地盘点、禁用、归档、删除积累的技能。

**产品名与品牌**（已确认）：

- 包/命令名：`skim`
- 显示名：**Skim** —— 含义"略读/撇去浮沫"贴合"快速盘点 + 清理冗余"
- 仓库目录 `packages/skm` 建议择机重命名为 `packages/skim` 保持一致

**UI 语言**（建议决策，待确认）：英文为默认语言（公开发布面向全球生态），内置 i18n 框架并首发附带 zh-CN —— 你自己是种子用户，中文不能等 v2。

## 2. 目标与非目标

### 目标（MVP 衡量口径）

| # | 目标 | 衡量 |
|---|---|---|
| G1 | 零配置开箱即见 | 首次启动 ≤ 3 秒内呈现全部用户级技能，无需任何设置 |
| G2 | 安全感拉满 | 所有破坏性操作可逆（原生配置可回切 / 归档可恢复 / 废纸篓可捞回），0 条"误删无法找回"反馈 |
| G3 | 大扫除效率 | 清理 30+ 技能（浏览→批量决策→执行）在 5 分钟内完成 |
| G4 | 不破坏用户现场 | 写配置保留用户既有内容与注释；卸载 Skim 后 agent 行为不被破坏 |

### 非目标（MVP 明确不做）

- 不做技能的创建/编辑（与 skill-creator 类工具不竞争）
- 不做技能市场、下载安装、版本更新
- 不做 plugin 管理（Claude 已有 `enabledPlugins` 官方机制）
- 不做健康检查 lint、使用频率统计、跨 agent 同步（v2 backlog）
- 不做 Windows/Linux 首发打磨（架构跨平台，首发 macOS，其余社区反馈驱动）

## 3. 用户画像与核心场景

**P1 普通 AI 工具用户（首要）**：用 Claude Code / Codex 但不熟终端深操作；装过别人分享的技能包；隐约觉得"助手变慢/变怪"但不知道怎么清理。
**P2 重度技能玩家（次要）**：自己写技能、装了几十个；关心上下文占用；需要四档精细控制和批量操作。

**核心场景**

1. **大扫除**：P1 周末打开 Skim → 看到 67 个技能（32+31+4）→ 按"重复描述"提示发现三个 test skill → 多选 → 删除（进废纸篓）→ 列表清爽
2. **临时关闭**：P2 发现某技能频繁误触发 → 搜索定位 → 切到 `off` → 下次会话生效；一周后想用了再切回
3. **上下文瘦身**：P2 想保留技能但减少系统提示占用 → 高级模式把低频技能批量切到 `name-only`
4. **留底封存**：P1 某项目结束 → 把项目技能归档到中央区 → 半年后新项目要用 → 从归档区一键恢复

## 5. 功能需求详述

### F1 扫描与列表（US-1, US-7）

- 启动即扫描：`~/.claude/skills`、`~/.codex/skills`、`~/.agents/skills` + 已发现项目的 `.claude/skills`、`.agents/skills`
- 项目自动发现：解析 `~/.claude/projects/` 目录名编码、`~/.codex/session_index.jsonl`；**校验目录真实存在，不存在不展示**；按含技能数排序
- 每个技能解析 frontmatter（name/description）；解析失败的技能仍展示，标"⚠ 元数据异常"
- 重复检测：同名或同 description 的技能行内打"重复"徽章
- 刷新：手动刷新按钮 + 定时自动刷新（默认开，间隔可配置，默认 30s，仅在窗口前台时执行）
- Codex bundled system skills（`.system/`）：展示 + 可禁用（原生机制支持），但**不可归档/删除**（agent 自带资产）

### F2 详情（US-2）

- Markdown 渲染 SKILL.md（代码块高亮；相对路径资源不必渲染）
- frontmatter 键值表；文件树（懒加载）；目录总体积
- "在 Finder 中打开"（跨平台对应文件管理器）

### F3 状态控制（US-3）

- **默认模式**：每行一个开关（on ⇄ off）
- **高级模式**（设置开启后）：Claude 技能显示四档选择器 `on / name-only / user-invocable-only / off`；Codex 技能为 on/off + `allow_implicit_invocation` 开关（对应"禁止自动触发"）
- 写入目标：
  - Claude 用户级技能 → `~/.claude/settings.json` 的 `skillOverrides`
  - Claude 项目级技能 → `<project>/.claude/settings.local.json` 的 `skillOverrides`
  - Codex 任意技能 → `~/.codex/config.toml` 的 `[[skills.config]]`（绝对路径）
- **配置写入三原则**：① 只增改自己管理的键，其余内容与注释原样保留；② 每次写入前在 `~/.skim/backups/` 留时间戳备份（保留最近 N 份）；③ 外部手工改过的状态，Skim 刷新后如实读出（配置是唯一事实源，Skim 不另存状态副本）
- 状态变更后提示："对新会话生效"（Codex 额外提示需重启会话）

### F4 归档（US-4）

- 归档 = 移动目录到 `~/.skim/archive/<agent>/<源路径哈希>/<skill-name>/` + 写 `manifest.json`（源绝对路径、agent、scope、归档时间、原状态）
- 归档原样存目录、不压缩（可浏览、可被 Spotlight 搜索；体积不是瓶颈——技能均值 < 1MB）→ §8 遗留议题 4 就此关闭
- 归档时若该技能有禁用配置，**一并清除该配置项**（避免残留死配置）
- 恢复：还原到源路径；源路径已存在同名 → 弹冲突对话框（覆盖 / 保留两者-重命名 / 取消）；源路径父目录已不存在 → 允许选择新位置
- 归档区支持"彻底删除"（进废纸篓）

### F5 删除（US-5）

- 一律进系统废纸篓；确认框展示：技能名、agent、路径、体积
- 同归档：连带清除其禁用配置项
- 批量删除确认框逐项列出清单

### F6 批量操作（US-6）

- 批量模式：复选框 + 按当前筛选结果全选
- 支持批量：disable / enable / 切档（高级模式）/ 归档 / 删除
- 执行前摘要确认；执行后逐项结果反馈，失败项可重试；部分失败不回滚成功项

## 6. 关键边界与异常

| 场景 | 行为 |
|---|---|
| agent 配置文件不存在（如从未装 Codex） | 对应 agent 分组整体隐藏，不报错 |
| 配置文件解析失败（损坏 TOML/JSON） | 该 agent 状态列降级为"只读 + ⚠ 配置异常"，禁止写入，引导用户检查 |
| 技能目录无 SKILL.md（游离文件，如 `rams.md`） | 列表中标"游离文件"，仅支持删除 |
| Skim 运行中外部删除了技能目录 | 下次刷新自动消失；操作时目标不存在 → 友好报错并刷新 |
| 两个 agent 目录通过软链指向同一实体 | 解析真实路径去重展示，标注"被多个 agent 共享" |
| 磁盘权限不足（macOS 完全磁盘访问） | 首次扫描失败时给出系统设置引导页 |

## 7. 非功能需求

- **性能**：200 个技能全量扫描 < 1s；UI 操作响应 < 100ms；定时刷新不可见卡顿
- **安全**：永不 `rm -rf`；所有写操作前备份；Tauri capabilities 限定可访问路径为 `$HOME/.claude/**`、`$HOME/.codex/**`、`$HOME/.agents/**`、`$HOME/.skim/**` + 用户显式添加的项目目录
- **平台**：macOS 首发（13+），Windows/Linux 构建可用但不阻塞发布
- **更新**：MVP 不做应用内自动更新，靠 Homebrew cask / 手动；版本检查提示可有
- **遥测**：零遥测、零网络请求（隐私是卖点，README 明示）
